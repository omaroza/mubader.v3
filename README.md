# مبادر كوتشينج — النسخة الثالثة (تخزين حقيقي يعمل على كل الأجهزة)

## ما الجديد في هذه النسخة

هذه المرة، أي تعديل تقوم به من لوحة التحكم (السعر، إضافة ورشة، صورة الكوتش، الشعار، الطلبات، السلات المتروكة) **يُحفظ في قاعدة بيانات حقيقية على الإنترنت** ويظهر فوراً لكل زائر من أي جهاز — كمبيوتر، جوال، أي متصفح. لم يعد التخزين محصوراً بجهازك فقط.

هذا يستخدم خدمة اسمها **Vercel Blob**، وهي مدمجة مع Vercel نفسه (نفس المكان الذي ينشر فيه موقعك)، فلا تحتاج حساباً منفصلاً أو كلمة مرور إضافية — فقط خطوة تفعيل واحدة سأشرحها بالأسفل.

## الفرق الجوهري عن النسخة السابقة

النسخة السابقة كانت ملفي HTML فقط (`index.html` و `admin.html`). هذه النسخة **مشروع برمجي متكامل** يحتوي على:

- `public/index.html` و `public/admin.html` — نفس الصفحتين، لكن بمنطق جديد يتواصل مع خادم بدلاً من التخزين المحلي
- `api/data.js` — نقطة اتصال (API) تخزن وتسترجع كل بيانات الموقع (المنتجات، الطلبات، السلات...)
- `api/upload.js` — نقطة اتصال لرفع الصور (صورة الكوتش والشعار)
- `package.json`, `package-lock.json`, `vercel.json` — ملفات إعداد المشروع، لا تحتاج لمسها إطلاقاً
- `.gitignore` — يخبر Git أي ملفات لا يجب رفعها

**هذا يعني أن طريقة الرفع على GitHub تتغيّر قليلاً** — بدل رفع ملفين فقط، سترفع **المجلد كاملاً بكل ما فيه** (راجع خطوات الرفع بالأسفل).

## بيانات الدخول للوحة التحكم

- الرابط: `your-site.vercel.app/admin.html`
- اسم المستخدم: `admin`
- كلمة المرور: `mubadir2025`

⚠️ غيّرها فور الدخول من: الإعدادات والأمان ← تغيير كلمة المرور (هذه تبقى محفوظة على جهازك فقط، وهذا مقصود ولا مشكلة فيه).

## خطوات الرفع والتفعيل (المرة الأولى فقط)

### الخطوة 1 — ارفع المشروع كاملاً على GitHub

هذه المرة لا ترفع ملفين فقط، بل **كل محتويات هذا المجلد** بما فيها مجلد `api` ومجلد `public` بالكامل، مع الحفاظ على نفس أسماء المجلدات بالضبط:

1. أنشئ مستودع GitHub جديد (أو استخدم مستودعك الحالي إن أردت البدء من جديد)
2. ارفع جميع الملفات والمجلدات كما هي: `api/`, `public/`, `package.json`, `package-lock.json`, `vercel.json`, `.gitignore`
3. **Commit changes**

### الخطوة 2 — اربط المستودع بـ Vercel

1. افتح **vercel.com** ← **Add New** ← **Project**
2. اختر **Import Git Repository** واختر المستودع الذي رفعته
3. اضغط **Deploy** (لا تغيّر أي إعدادات افتراضية، Vercel سيتعرف على المشروع تلقائياً)

في هذه المرحلة، الموقع سينشر لكنه **لن يعمل بشكل كامل بعد** لأن قاعدة البيانات (Vercel Blob) لم تُفعّل بعد — هذه هي الخطوة التالية والأهم.

### الخطوة 3 — تفعيل Vercel Blob (مرة واحدة فقط)

1. من داخل مشروعك في Vercel، اضغط على تبويب **Storage** من القائمة العلوية
2. اضغط **Create Database** أو **Connect Store**
3. اختر **Blob**
4. اعطه اسماً (مثلاً `mubadir-storage`) واضغط **Create**
5. Vercel سيسألك ربط هذا التخزين بمشروعك — اضغط **Connect** أو **Connect Project**
6. اختر مشروعك من القائمة واضغط **Connect**

بمجرد الربط، Vercel يضيف تلقائياً متغير بيئة يسمى `BLOB_READ_WRITE_TOKEN` لمشروعك — **لا تحتاج نسخه أو كتابته يدوياً**، هذا يحدث تلقائياً بالكامل.

### الخطوة 4 — أعد النشر (Redeploy)

بعد ربط Blob، يجب إعادة نشر الموقع مرة واحدة حتى يلتقط متغير البيئة الجديد:

1. من تبويب **Deployments**
2. اضغط **...** بجانب آخر نشر ← **Redeploy**
3. اضغط **Redeploy** للتأكيد

### الخطوة 5 — اختبر

افتح موقعك من الكمبيوتر وغيّر شيئاً من لوحة التحكم (مثلاً السعر)، ثم افتح نفس الموقع من الهاتف — يجب أن ترى التغيير فوراً بدون أي إعادة رفع أو خطوات إضافية.

## كيف تحدّث الموقع لاحقاً (بعد الإعداد الأولي)

من الآن فصاعداً، التحديثات نوعان:

**تحديثات المحتوى (سعر، منتجات، صور، نصوص):** تتم بالكامل من لوحة التحكم مباشرة، وتظهر فوراً لكل الزوار. لا حاجة لـ GitHub أو Vercel إطلاقاً لهذا النوع.

**تحديثات الكود نفسه (تصميم جديد، صفحة جديدة، خلل تقني):** فقط في هذه الحالة، ترفع الملفات المُعدّلة على GitHub بنفس الطريقة المعتادة، و Vercel ينشرها تلقائياً.
